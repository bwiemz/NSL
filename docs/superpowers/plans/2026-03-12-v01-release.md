- [ ] **Step 2: Create formatter.rs — indentation safety**

Create `crates/nsl-cli/src/formatter.rs`. Start with the indentation safety check:

```rust
pub struct FormatResult {
    pub changed: bool,
    pub output: String,
}

pub fn format_source(input: &str) -> Result<FormatResult, String> {
    // Phase 1: Check for mixed indentation
    check_indentation_safety(input)?;

    let mut output = input.to_string();

    // Phase 2: Convert tabs to 4 spaces (if file uses tabs)
    output = normalize_tabs(&output);

    // Phase 3: Trailing whitespace
    output = strip_trailing_whitespace(&output);

    // Phase 4: Blank lines (max 2 consecutive)
    output = normalize_blank_lines(&output);

    // Phase 5: Operator spacing
    output = normalize_operators(&output);

    // Phase 6: String quotes (single → double)
    output = normalize_quotes(&output);

    let changed = output != input;
    Ok(FormatResult { changed, output })
}
```

The `check_indentation_safety` function scans each line. If any line has BOTH tabs and spaces as leading whitespace, return error:

```rust
fn check_indentation_safety(input: &str) -> Result<(), String> {
    for (i, line) in input.lines().enumerate() {
        let leading: String = line.chars().take_while(|c| c.is_whitespace()).collect();
        let has_tabs = leading.contains('\t');
        let has_spaces = leading.contains(' ');
        if has_tabs && has_spaces {
            return Err(format!(
                "error: ambiguous mixed indentation at line {}. Fix manually before formatting.",
                i + 1
            ));
        }
    }
    Ok(())
}
```

- [ ] **Step 3: Implement formatting passes**

Each pass is a simple function:

- `normalize_tabs`: Replace all `\t` in leading whitespace with 4 spaces
- `strip_trailing_whitespace`: For each line, trim trailing spaces/tabs
- `normalize_blank_lines`: Collapse runs of 3+ blank lines to 2
- `normalize_operators`: Regex-based: ensure single space around `+`, `-`, `*`, `/`, `=`, `==`, `!=`, `<=`, `>=`, `|>` etc. Be careful with unary minus (preceded by `=`, `(`, `,`, `[`, or start-of-line)
- `normalize_quotes`: Replace `'string'` with `"string"` (only outside of already-double-quoted strings and comments)

**Note on import sorting:** Per the spec, defer import sorting to v0.1.1 if it proves complex. It requires light parsing to identify import blocks, which is more than pure line processing. Implement the other passes first.

3. If `check` mode: report which files need formatting, exit 1 if any changed
4. If write mode: write formatted output back to file
5. Print summary: `Formatted N files (M changed)`

```rust
//! End-to-end smoke tests for NSL examples.
//! Compiles and runs .nsl files, compares stdout against expected baselines.

use std::path::{Path, PathBuf};
use std::process::Command;

/// Truncate floating-point numbers in text to 4 decimal places.
fn normalize_floats(text: &str) -> String {
    let re = regex::Regex::new(r"-?\d+\.\d{5,}").unwrap();
    re.replace_all(text, |caps: &regex::Captures| {
        let val: f64 = caps[0].parse().unwrap_or(0.0);
        format!("{:.4}", val)
    }).to_string()
}

/// Replace absolute paths with <PATH>.
fn normalize_paths(text: &str) -> String {
    let re = regex::Regex::new(r"[A-Z]:\\[^\s]+|/(?:home|Users|tmp)/[^\s]+").unwrap();
    re.replace_all(text, "<PATH>").to_string()
}

fn normalize(text: &str) -> String {
    normalize_paths(&normalize_floats(text))
}

fn nsl_binary() -> PathBuf {
    // cargo test builds to target/debug/
    let mut path = std::env::current_exe().unwrap();
    path.pop(); // remove test binary name
    path.pop(); // remove deps/
    path.push("nsl-cli"); // or just "nsl" depending on binary name
    // Fallback: use cargo run
    path
}

fn run_example(name: &str) -> String {
    let example_path = format!("examples/{}.nsl", name);
    let output = Command::new("cargo")
        .args(["run", "-p", "nsl-cli", "--", "run", &example_path])
        .output()
        .expect("failed to execute nsl run");
    String::from_utf8_lossy(&output.stdout).to_string()
}

fn expected_output(name: &str) -> String {
    let path = format!("tests/expected/{}.txt", name);
    std::fs::read_to_string(&path)
        .unwrap_or_else(|_| panic!("Missing baseline: {}", path))
}

fn assert_output_matches(name: &str) {
    let actual = normalize(&run_example(name));
    let expected = normalize(&expected_output(name));
    assert_eq!(
        actual.trim(),
        expected.trim(),
        "Output mismatch for {}",
        name
    );
}

#[test] fn test_hello() { assert_output_matches("hello"); }
#[test] fn test_features() { assert_output_matches("features"); }
#[test] fn test_m9_tensors() { assert_output_matches("m9_tensors"); }
#[test] fn test_m10_shape_check() { assert_output_matches("m10_shape_check"); }
#[test] fn test_m11_model_basic() { assert_output_matches("m11_model_basic"); }
#[test] fn test_m12_grad_basic() { assert_output_matches("m12_grad_basic"); }
#[test] fn test_m12_grad_matmul() { assert_output_matches("m12_grad_matmul"); }
#[test] fn test_m13_stdlib_import() { assert_output_matches("m13_stdlib_import"); }
#[test] fn test_m14_sgd_basic() { assert_output_matches("m14_sgd_basic"); }
```

- [ ] **Step 1: Add runtime-relative search to linker.rs**

In `linker.rs`, modify the runtime library path resolution. Currently it uses:

```rust
const RUNTIME_LIB_PATH: &str = env!("NSL_RUNTIME_LIB_PATH");
```

Change to a function that searches multiple locations:

```rust
fn find_runtime_lib() -> PathBuf {
    // 1. Check NSL_RUNTIME_LIB_PATH env var (development builds)
    if let Ok(path) = std::env::var("NSL_RUNTIME_LIB_PATH_OVERRIDE") {
        let p = PathBuf::from(&path);
        if p.exists() { return p; }
    }

    // 2. Check relative to executable: <exe>/../lib/
    if let Ok(exe) = std::env::current_exe() {
        let lib_dir = exe.parent().unwrap().parent().unwrap().join("lib");
        let lib_name = if cfg!(windows) { "nsl_runtime.lib" } else { "libnsl_runtime.a" };
        let lib_path = lib_dir.join(lib_name);
        if lib_path.exists() { return lib_path; }
    }

    // 3. Fallback to compile-time path (cargo build scenario)
    let compile_time = PathBuf::from(env!("NSL_RUNTIME_LIB_PATH"));
    if compile_time.exists() { return compile_time; }

    panic!("Cannot find nsl runtime library. Ensure the lib/ directory is next to bin/.");
}
```

Then use `find_runtime_lib()` instead of the const in both `link()` and `link_multi()`.

```yaml
    tags: ['v*']

permissions:
  contents: write

jobs:
  build-release:
    strategy:
      matrix:
        include:
          - os: ubuntu-latest
            target: x86_64-unknown-linux-gnu
            archive: tar.gz
          - os: windows-latest
            target: x86_64-pc-windows-msvc
            archive: zip
          - os: macos-14
            target: aarch64-apple-darwin
            archive: tar.gz
          - os: macos-13
            target: x86_64-apple-darwin
            archive: tar.gz
    runs-on: ${{ matrix.os }}

    steps:
      - uses: actions/checkout@v4

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable
        with:
          targets: ${{ matrix.target }}

      - name: Build release binary
        run: cargo build --release --target ${{ matrix.target }} -p nsl-cli

      - name: Build runtime library
        run: cargo build --release --target ${{ matrix.target }} -p nsl-runtime

      - name: Package toolchain (Unix)
        if: matrix.archive == 'tar.gz'
        run: |
          TAG=${GITHUB_REF#refs/tags/}
          DIR="nsl-${TAG}-${{ matrix.target }}"
          mkdir -p "${DIR}/bin" "${DIR}/lib"
          cp "target/${{ matrix.target }}/release/nsl-cli" "${DIR}/bin/nsl"
          cp "target/${{ matrix.target }}/release/libnsl_runtime.a" "${DIR}/lib/"
          cp -r stdlib "${DIR}/lib/stdlib"
          cp LICENSE README.md "${DIR}/"
          tar czf "${DIR}.tar.gz" "${DIR}"

      - name: Package toolchain (Windows)
        if: matrix.archive == 'zip'
        shell: pwsh
        run: |
          $TAG = $env:GITHUB_REF -replace 'refs/tags/',''
          $DIR = "nsl-${TAG}-${{ matrix.target }}"
          New-Item -ItemType Directory -Path "${DIR}/bin","${DIR}/lib" -Force
          Copy-Item "target/${{ matrix.target }}/release/nsl-cli.exe" "${DIR}/bin/nsl.exe"
          Copy-Item "target/${{ matrix.target }}/release/nsl_runtime.lib" "${DIR}/lib/"
          Copy-Item -Recurse "stdlib" "${DIR}/lib/stdlib"
          Copy-Item LICENSE,README.md "${DIR}/"
          Compress-Archive -Path "${DIR}" -DestinationPath "${DIR}.zip"

      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: nsl-${{ matrix.target }}
          path: nsl-*.${{ matrix.archive }}

  create-release:
    needs: build-release
    runs-on: ubuntu-latest
    steps:
      - uses: actions/download-artifact@v4
        with:
          merge-multiple: true

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          files: nsl-*
          generate_release_notes: true
```

```markdown
# Changelog

All notable changes to NeuralScript will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/).

## [0.1.0] - 2026-03-XX

### Language Features
- Indentation-based syntax with Python-familiar keywords
- Pipe operator (`|>`) for model op chaining
- `let`/`const` variable declarations with type inference
- `fn` functions with named/default parameters
- `model` keyword for neural network definitions
- `grad` keyword for tape-based automatic differentiation
- `train` block DSL with declarative data/optimizer/scheduler
- `quant` block for INT4/INT8 weight quantization
- `kernel` keyword for custom GPU kernels (PTX codegen)
- Compile-time tensor shape checking with named dimensions
- `@no_grad`, `@checkpoint`, `@backward`, `@test` decorators
- Import system with multi-file compilation

### Standard Library
- **nsl.nn**: Linear, Embedding, Conv2d, MaxPool2d, LayerNorm, RMSNorm, Dropout, Attention, TransformerBlock
- **nsl.nn.activations**: relu, gelu, silu, sigmoid, tanh, softmax, elu
- **nsl.nn.losses**: mse_loss, l1_loss, cross_entropy, bce_loss
- **nsl.optim**: SGD, Adam, AdamW, Lion, Muon, SOAP
- **nsl.optim.schedulers**: constant_lr, step_lr, exponential_lr, linear_decay, cosine_anneal, warmup_cosine, one_cycle
- **nsl.tokenize**: byte_tokenizer, BPE encode/decode
- **nsl.data**: JSONL/CSV/mmap DataLoader with batching, shuffling, sequence packing
- **nsl.inference**: topk, multinomial, argmax, autoregressive generation
- **nsl.quant**: quantize/dequantize (INT4/INT8)
- **nsl.compat**: safetensors load/save, HuggingFace model loading, ONNX export

### Tooling
- `nsl run` — compile and execute NSL programs
- `nsl build` — compile to native executable
- `nsl check` — type checking and semantic analysis
- `nsl test` — run `@test` annotated functions
- `nsl export` — ONNX model export
- `nsl fmt` — code formatter
- `nsl init` — project scaffolding

### GPU Support
- CUDA backend with 15 PTX kernels (elementwise ops + matmul)
- `kernel` keyword for custom GPU ops
- Device transfer (`.to(cuda)`, `.to(cpu)`)
- Unified memory via cuMemAllocManaged

### Interop
- Safetensors read/write
- HuggingFace Hub model loading (single + sharded)
- ONNX export

### Known Limitations
- No package manager or dependency resolution
- No PyTorch FFI (`to_torch`/`from_torch`)
- No distributed multi-GPU training (DDP)
- No REPL
- CUDA required for GPU features (no ROCm/Metal)
- Windows requires Visual Studio Build Tools for linking
```
