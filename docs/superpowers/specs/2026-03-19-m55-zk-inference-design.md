# M55: Zero-Knowledge Inference Circuits — Design Specification

**Date:** 2026-03-19
**Status:** Planned
**Milestone:** M55 (Phase 11, v1.0.0)
**Prerequisites:** M52 (Weight-Aware Compilation — static DAG with known weights), M12 (Autodiff/Tape — Wengert list), M35 (FP8/INT8 Quantization — field-compatible arithmetic)
**Dependencies:** M52 provides the weight-aware static DAG that makes circuit emission tractable; M35's INT8 quantized models map naturally to finite field elements; M40 (Source-to-Source AD) Wengert list is the ideal IR for circuit translation

## Overview

M55 compiles a model forward pass into a zero-knowledge proof circuit, enabling a prover to demonstrate that a specific output was generated by a specific model architecture — without revealing the model weights, the input, or both. This is the feature that makes NSL the default language for verifiable AI: model providers can prove inference integrity to clients, regulators can verify that a deployed model matches a certified architecture, and on-chain AI oracles can provide cryptographically verified predictions.

The key design insight is that NSL's static, shape-checked expression DAG (especially after M52's weight-aware constant folding) is the ideal source for arithmetic circuit compilation. Every tensor operation is statically known — there are no dynamic branches, no data-dependent control flow, no Python `if` statements that change the computation graph at runtime. The Wengert list from M12/M40 is already a flat sequence of arithmetic operations with explicit data dependencies — it maps directly to R1CS (Rank-1 Constraint System) constraints. INT8 quantized models (M35) operate in small integer arithmetic that fits naturally within the scalar field of BN254 or BLS12-381 elliptic curves.

The `@zk_proof` decorator marks a model forward function for circuit compilation. The compiler translates the expression DAG into an R1CS arithmetic circuit, where each tensor operation becomes a set of field-element constraints. The Halo2 proving backend (pure Rust, no C/C++ dependencies) generates proofs and verification keys. A Solidity verification contract is optionally emitted for on-chain verification. The `nsl build --zk-circuit model.nsl` CLI produces three artifacts: the circuit description, the proving key, and the verification key.

**Why this is impossible in Python:** Python's dynamic control flow means the computation graph is not known until runtime. Generating ZK circuits from traced PyTorch graphs (e.g., EZKL, Modulus Labs) requires re-tracing for every input shape and architecture change, produces circuits 10-100x larger than necessary due to unresolved dynamic dispatch, and cannot handle model-weight privacy (the tracer sees the weights). NSL's static DAG, combined with M52's weight-aware compilation, produces minimal circuits with weight privacy built in.

## Section 1: Language Surface

### 1.1 `@zk_proof` Decorator

```
@zk_proof
fn forward(self, x: Tensor<[1, 784], int8>) -> Tensor<[1, 10], int8>:
    let h1 = relu(matmul(x, self.W1) + self.b1)
    let h2 = relu(matmul(h1, self.W2) + self.b2)
    return matmul(h2, self.W3) + self.b3
```

**`@zk_proof` guarantees:**
- The function body contains only circuit-compatible operations (see Section 1.4)
- No dynamic control flow (no data-dependent `if`, no variable-length loops)
- No floating-point arithmetic (must use INT8 or fixed-point — see Section 1.3)
- No in-place mutation (pure function — composable with M51's `@pure`)
- No IO or communication effects

**`@zk_proof` produces:**
- R1CS constraint system representing the computation
- Witness generation function (maps concrete inputs/weights to field elements)
- Verification key (public, given to verifier)
- Proving key (private, used by prover)

### 1.2 Proof Modes

```
# Mode 1: Prove output integrity — public input, public architecture, private weights
@zk_proof(mode="weight_private")
fn forward(self, x: Tensor<[1, 784], int8>) -> Tensor<[1, 10], int8>:
    ...

# Mode 2: Prove input integrity — private input, public weights, public output
@zk_proof(mode="input_private")
fn forward(self, x: Tensor<[1, 784], int8>) -> Tensor<[1, 10], int8>:
    ...

# Mode 3: Full privacy — private input AND private weights, public output only
@zk_proof(mode="full_private")
fn forward(self, x: Tensor<[1, 784], int8>) -> Tensor<[1, 10], int8>:
    ...

# Mode 4: Architecture attestation — prove output comes from THIS architecture
#          without revealing weights or input (default)
@zk_proof
fn forward(self, x: Tensor<[1, 784], int8>) -> Tensor<[1, 10], int8>:
    ...
```

The `mode` parameter controls which values are public (part of the verification statement) and which are private (known only to the prover):

| Mode | Input | Weights | Output | Architecture |
|------|-------|---------|--------|-------------|
| `weight_private` | Public | Private | Public | Public |
| `input_private` | Private | Public | Public | Public |
| `full_private` | Private | Private | Public | Public |
| (default) | Private | Private | Public | Public (embedded in circuit) |

### 1.3 Fixed-Point Arithmetic

ZK circuits operate over finite fields (large prime numbers). Floating-point arithmetic does not map to field operations. INT8 quantized models (M35) are the natural fit: each int8 value is a field element, and int8 addition/multiplication maps directly to field addition/multiplication (with overflow checks as constraints).

```
# This works — INT8 arithmetic maps to field elements
model TinyMLP:
    W1: Tensor<[784, 128], int8>
    b1: Tensor<[128], int8>
    W2: Tensor<[128, 10], int8>
    b2: Tensor<[10], int8>

    @zk_proof
    fn forward(self, x: Tensor<[1, 784], int8>) -> Tensor<[1, 10], int8>:
        let h = relu(matmul(x, self.W1) + self.b1)
        return matmul(h, self.W2) + self.b2
```

```
# This does NOT work — float arithmetic is not field-compatible
model FloatMLP:
    W1: Tensor<[784, 128], f32>

    @zk_proof    # ERROR: @zk_proof requires integer or fixed-point types
    fn forward(self, x: Tensor<[1, 784], f32>) -> Tensor<[1, 128], f32>:
        return matmul(x, self.W1)
```

For models originally trained in floating-point, the workflow is:
1. Train in f32/f16
2. Quantize to INT8 using M35's quantization infrastructure
3. Apply `@zk_proof` to the quantized model

The compiler also supports a built-in `FixedPoint<BITS, FRAC>` type for higher precision:

```
# Fixed-point: 16-bit total, 8 fractional bits (range: -128.0 to 127.996)
@zk_proof
fn forward(self, x: Tensor<[1, 784], FixedPoint<16, 8>>) -> Tensor<[1, 10], FixedPoint<16, 8>>:
    let h = relu(fixed_matmul(x, self.W1) + self.b1)
    return fixed_matmul(h, self.W2) + self.b2
```

### 1.4 Circuit-Compatible Operations

Not all NSL operations can be expressed as arithmetic circuits. The `@zk_proof` decorator restricts the function body to the following operations:

**Allowed operations (circuit-compatible):**

| Operation | Constraint Cost | Notes |
|-----------|----------------|-------|
| `matmul(A, B)` | O(m * n * k) mul constraints | Core operation |
| `a + b` (element-wise) | O(n) add constraints | Free in R1CS (linear combination) |
| `a - b` (element-wise) | O(n) add constraints | Free in R1CS |
| `a * b` (element-wise) | O(n) mul constraints | One constraint per element |
| `relu(x)` | O(n) comparison constraints | Implemented as bit decomposition |
| `max(a, b)` | O(n) comparison constraints | Bit decomposition |
| `reshape(x, shape)` | 0 constraints | Wire permutation only |
| `transpose(x)` | 0 constraints | Wire permutation only |
| `concat(a, b)` | 0 constraints | Wire concatenation |
| `slice(x, ...)` | 0 constraints | Wire selection |
| `sum(x, dim)` | O(n) add constraints | Free in R1CS |
| Scalar constants | 0 constraints | Embedded in circuit |

**Disallowed operations (not circuit-compatible):**

| Operation | Why Disallowed |
|-----------|---------------|
| `softmax(x)` | Requires division and exp() — not field-compatible |
| `exp(x)`, `log(x)` | Transcendental functions have no field representation |
| `sqrt(x)` | Requires inverse square root — expensive in circuits |
| `dropout(x)` | Randomness is not compatible with deterministic proofs |
| `layer_norm(x)` | Requires division by standard deviation |
| Data-dependent `if` | Circuits are static — no branching |
| Variable-length loops | Circuit size must be known at compile time |

```
error[E0550]: operation 'softmax' is not compatible with @zk_proof
  --> model.nsl:8:16
   |
 8 |     let probs = softmax(logits, dim=-1)
   |                 ^^^^^^^ softmax requires exp() and division
   |
   = note: ZK circuits operate over finite fields — transcendental functions have no field representation
   = help: use argmax(logits) for classification, or implement a lookup-table approximation
   = help: see docs/zk-compatible-ops.md for circuit-compatible alternatives
```

### 1.5 CLI Interface

```bash
# Compile model to ZK circuit
nsl build --zk-circuit model.nsl --weights model.safetensors

# Specify proving backend
nsl build --zk-circuit model.nsl --zk-backend halo2   # default
nsl build --zk-circuit model.nsl --zk-backend plonky3

# Generate Solidity verification contract
nsl build --zk-circuit model.nsl --zk-solidity

# Generate proof for a specific input
nsl zk prove model.circuit --input input.json --output proof.bin

# Verify a proof
nsl zk verify model.vk --proof proof.bin --public-inputs output.json

# Circuit statistics (constraint count, proof size estimate)
nsl zk stats model.circuit
```

**Build artifacts:**

| File | Description |
|------|-------------|
| `model.circuit` | Serialized R1CS constraint system |
| `model.pk` | Proving key (private — prover only) |
| `model.vk` | Verification key (public — shared with verifiers) |
| `model_verifier.sol` | Solidity contract for on-chain verification (with `--zk-solidity`) |

### 1.6 Proof Generation and Verification in NSL

```
# Generate and verify proof programmatically
let circuit = zk.load_circuit("model.circuit")
let pk = zk.load_proving_key("model.pk")
let vk = zk.load_verification_key("model.vk")

# Prover side
let input = tensor([1, 0, 0, ...], dtype=int8)  # 784-element input
let proof = zk.prove(circuit, pk, input=input, weights=model.weights)
let public_output = proof.public_output            # the model's prediction

# Verifier side (only needs vk, proof, and public_output)
let valid = zk.verify(vk, proof, public_output)
assert(valid)  # cryptographically guaranteed: output came from this architecture
```

### 1.7 Integration with M52 Weight-Aware Compilation

When `--weights` is provided alongside `--zk-circuit`, M52's weight-aware pass runs first:

1. M52 loads weights and propagates known values through the DAG
2. Dead weight elimination removes near-zero weights (fewer constraints)
3. Sparsity-aware codegen identifies zero-weight positions (skip those mul constraints)
4. The ZK circuit compiler receives a simplified DAG with constants folded

This produces significantly smaller circuits: a model with 40% weight sparsity produces ~40% fewer multiplication constraints.

```rust
/// A variable in the arithmetic circuit.
/// Maps to a "wire" in R1CS terminology.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct Wire(pub u64);

/// A linear combination of wires: c_0 + c_1*w_1 + c_2*w_2 + ...
/// Represents one side of an R1CS constraint.
#[derive(Debug, Clone)]
pub struct LinearCombination {
    pub terms: Vec<(FieldElement, Wire)>,
    pub constant: FieldElement,
}

/// A single R1CS constraint: A * B = C
/// where A, B, C are linear combinations of wires.
#[derive(Debug, Clone)]
pub struct R1CSConstraint {
    pub a: LinearCombination,
    pub b: LinearCombination,
    pub c: LinearCombination,
    /// Source location for debugging
    pub origin: ConstraintOrigin,
}

/// The complete R1CS constraint system for a model.
#[derive(Debug)]
pub struct R1CSCircuit {
    /// Name of the model/function this circuit represents.
    pub name: String,

    /// Total number of wires (variables) in the circuit.
    pub num_wires: u64,

    /// Public input wires (visible to verifier).
    pub public_inputs: Vec<Wire>,

    /// Public output wires (visible to verifier).
    pub public_outputs: Vec<Wire>,

    /// Private input wires (known only to prover).
    /// Weights in weight_private mode, inputs in input_private mode.
    pub private_inputs: Vec<Wire>,

    /// All constraints: A_i * B_i = C_i
    pub constraints: Vec<R1CSConstraint>,

    /// Wire metadata for debugging.
    pub wire_names: HashMap<Wire, String>,

    /// Circuit statistics.
    pub stats: CircuitStats,
}
```

### 2.3 Field Element Representation

```rust
/// A finite field element for ZK arithmetic.
/// Uses the BN254 scalar field by default (prime ~254 bits).
/// INT8 values map directly: value v -> field element v (mod p).
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct FieldElement {
    /// Internal representation as 4x64-bit limbs (256 bits total).
    pub limbs: [u64; 4],
}

impl FieldElement {
    /// The BN254 scalar field prime:
    /// p = 21888242871839275222246405745257275088548364400416034343698204186575808495617
    pub const BN254_PRIME: FieldElement = FieldElement {
        limbs: [
            0x43e1f593f0000001,
            0x2833e84879b97091,
            0xb85045b68181585d,
            0x30644e72e131a029,
        ],
    };

    /// Convert an INT8 value to a field element.
    pub fn from_int8(val: i8) -> FieldElement {
        if val >= 0 {
            FieldElement { limbs: [val as u64, 0, 0, 0] }
        } else {
            // Negative values: p - |val| (mod p)
            Self::BN254_PRIME.sub(&FieldElement { limbs: [(-val) as u64, 0, 0, 0] })
        }
    }

    /// Convert a fixed-point value to a field element.
    /// FixedPoint<16, 8> with value 1.5 -> integer representation 384 (1.5 * 2^8)
    pub fn from_fixed_point(val: i64, frac_bits: u32) -> FieldElement {
        let scaled = val; // already scaled by caller
        if scaled >= 0 {
            FieldElement { limbs: [scaled as u64, 0, 0, 0] }
        } else {
            Self::BN254_PRIME.sub(&FieldElement { limbs: [(-scaled) as u64, 0, 0, 0] })
        }
    }

    pub fn zero() -> FieldElement {
        FieldElement { limbs: [0, 0, 0, 0] }
    }

    pub fn one() -> FieldElement {
        FieldElement { limbs: [1, 0, 0, 0] }
```

## Section 3: Circuit Compilation

### 3.1 Expression DAG to R1CS

The `CircuitCompiler` walks the expression DAG (from M12's Wengert list or directly from the typed AST) and emits R1CS constraints for each operation.

```rust
/// The main circuit compiler: walks the expression DAG and emits R1CS constraints.
pub struct CircuitCompiler<'a> {
    /// The R1CS circuit being built.
    circuit: R1CSCircuit,

    /// Next available wire ID.
    next_wire: u64,

    /// Maps AST variable names to circuit wires.
    var_to_wires: HashMap<Symbol, TensorWires>,

    /// Privacy mode (determines public/private classification).
    mode: ZkMode,

    /// Weight values (from M52's WeightMap, if available).
    weights: Option<&'a WeightMap>,

    /// Interner for symbol resolution.
    interner: &'a Interner,

    /// Source diagnostics.
    diagnostics: Vec<Diagnostic>,
}

/// Maps a tensor variable to its per-element wires in the circuit.
#[derive(Debug, Clone)]
pub struct TensorWires {
    /// One wire per element in the flattened tensor.
    pub wires: Vec<Wire>,
    /// Original tensor shape.
    pub shape: Vec<usize>,
    /// Whether this tensor is a public or private input.
    pub visibility: WireVisibility,
}

#[derive(Debug, Clone, Copy)]
pub enum WireVisibility {
    Public,
    Private,
    Intermediate,  // internal computation — not exposed
}
```

### 3.2 Operation-to-Constraint Mapping

```rust
impl<'a> CircuitCompiler<'a> {
    /// Compile a single expression node into R1CS constraints.
    fn compile_expr(&mut self, expr: &TypedExpr) -> TensorWires {
        match &expr.kind {
            // Addition: C = A + B
            // R1CS: (A_i + B_i) * 1 = C_i  for each element i
            // This is "free" — addition is a linear combination, costs zero constraints.
            TypedExprKind::BinaryOp { op: BinOp::Add, left, right } => {
                let a_wires = self.compile_expr(left);
                let b_wires = self.compile_expr(right);
                let c_wires = self.alloc_tensor_wires(&expr.shape, WireVisibility::Intermediate);

                for i in 0..a_wires.wires.len() {
                    // (a_i + b_i) * 1 = c_i
                    let a_lc = LinearCombination::single(FieldElement::one(), a_wires.wires[i]);
                    let b_lc = LinearCombination::single(FieldElement::one(), b_wires.wires[i]);
                    let a_plus_b = a_lc.add_lc(&b_lc);
                    let one = LinearCombination::constant(FieldElement::one());
                    let c_lc = LinearCombination::single(FieldElement::one(), c_wires.wires[i]);

                    self.circuit.add_constraint(R1CSConstraint {
                        a: a_plus_b,
                        b: one,
                        c: c_lc,
                        origin: self.origin(expr, i),
                    });
                }

                c_wires
            }

            // Multiplication (element-wise): C = A * B
            // R1CS: A_i * B_i = C_i  for each element i
            // Costs one constraint per element.
            TypedExprKind::BinaryOp { op: BinOp::Mul, left, right } => {
                let a_wires = self.compile_expr(left);
                let b_wires = self.compile_expr(right);
                let c_wires = self.alloc_tensor_wires(&expr.shape, WireVisibility::Intermediate);

                for i in 0..a_wires.wires.len() {
                    let a_lc = LinearCombination::single(FieldElement::one(), a_wires.wires[i]);
                    let b_lc = LinearCombination::single(FieldElement::one(), b_wires.wires[i]);
                    let c_lc = LinearCombination::single(FieldElement::one(), c_wires.wires[i]);

                    self.circuit.add_constraint(R1CSConstraint {
                        a: a_lc,
                        b: b_lc,
                        c: c_lc,
                        origin: self.origin(expr, i),
                    });
                }

                c_wires
            }

            // Matrix multiplication: C[i,j] = sum_k A[i,k] * B[k,j]
            // Each output element requires K multiplication constraints
            // plus K-1 addition constraints (free in R1CS).
            TypedExprKind::Call { func, args, .. }
                if self.interner.resolve(*func) == "matmul" =>
            {
                self.compile_matmul(&args[0], &args[1], &expr.shape)
            }

            // ReLU: out = max(x, 0)
            // Implemented as: x = x_pos - x_neg, x_pos * x_neg = 0, out = x_pos
            // where x_pos >= 0, x_neg >= 0 (enforced by bit decomposition).
            TypedExprKind::Call { func, args, .. }
                if self.interner.resolve(*func) == "relu" =>
            {
                self.compile_relu(&args[0], &expr.shape)
            }

            // Reshape/transpose: zero constraints, just rewire
            TypedExprKind::Call { func, args, .. }
                if self.interner.resolve(*func) == "reshape" =>
            {
                let input = self.compile_expr(&args[0]);
                TensorWires {
                    wires: input.wires,
                    shape: expr.shape.clone(),
                    visibility: WireVisibility::Intermediate,
                }
            }

            // Constant tensor (from weights or literal)
            TypedExprKind::Constant { value } => {
                self.compile_constant(value, &expr.shape)
            }

            // Model weight access: self.W1
            TypedExprKind::FieldAccess { object, field } => {
                self.compile_weight_access(object, field, &expr.shape)
            }

            _ => {
                self.diagnostics.push(Diagnostic::error(
                    format!("unsupported operation in @zk_proof: {:?}", expr.kind),
                    expr.span,
                ));
                self.alloc_tensor_wires(&expr.shape, WireVisibility::Intermediate)
            }
        }
    }

    /// Compile matmul: C[i,j] = sum_k A[i,k] * B[k,j]
    fn compile_matmul(
        &mut self,
        a_expr: &TypedExpr,
        b_expr: &TypedExpr,
        out_shape: &[usize],
    ) -> TensorWires {
        let a_wires = self.compile_expr(a_expr);
        let b_wires = self.compile_expr(b_expr);

        let m = a_wires.shape[0]; // rows of A
        let k = a_wires.shape[1]; // cols of A = rows of B
        let n = b_wires.shape[1]; // cols of B

        let c_wires = self.alloc_tensor_wires(out_shape, WireVisibility::Intermediate);

        for i in 0..m {
            for j in 0..n {
                // C[i,j] = sum_k A[i,k] * B[k,j]
                // We need intermediate wires for each product A[i,k] * B[k,j]
                let mut product_wires = Vec::with_capacity(k);

                for kk in 0..k {
                    let a_idx = i * k + kk;
                    let b_idx = kk * n + j;
                    let prod_wire = self.alloc_wire();
                    product_wires.push(prod_wire);

                    // Constraint: A[i,k] * B[k,j] = prod_wire
                    self.circuit.add_constraint(R1CSConstraint {
                        a: LinearCombination::single(FieldElement::one(), a_wires.wires[a_idx]),
                        b: LinearCombination::single(FieldElement::one(), b_wires.wires[b_idx]),
                        c: LinearCombination::single(FieldElement::one(), prod_wire),
                        origin: ConstraintOrigin {
                            source_span: Span::default(),
                            operation: format!("matmul[{},{}] += A[{},{}]*B[{},{}]", i, j, i, kk, kk, j),
                            tensor_indices: Some(vec![i, j, kk]),
                        },
                    });
                }

                // Sum constraint: sum(products) = C[i,j]
                // This is a linear combination — "free" in R1CS
                let c_idx = i * n + j;
                let sum_lc = LinearCombination {
                    terms: product_wires.iter().map(|&w| (FieldElement::one(), w)).collect(),
                    constant: FieldElement::zero(),
                };
                self.circuit.add_constraint(R1CSConstraint {
                    a: sum_lc,
                    b: LinearCombination::constant(FieldElement::one()),
                    c: LinearCombination::single(FieldElement::one(), c_wires.wires[c_idx]),
                    origin: ConstraintOrigin {
                        source_span: Span::default(),
                        operation: format!("matmul[{},{}] = sum", i, j),
                        tensor_indices: Some(vec![i, j]),
                    },
                });
            }
        }

        c_wires
    }

    /// Compile ReLU: out = max(x, 0)
    /// Standard ZK trick: decompose x into positive and negative parts.
    /// x = x_pos - x_neg, x_pos >= 0, x_neg >= 0, x_pos * x_neg = 0
    /// Then out = x_pos.
    fn compile_relu(&mut self, input_expr: &TypedExpr, out_shape: &[usize]) -> TensorWires {
        let x_wires = self.compile_expr(input_expr);
        let out_wires = self.alloc_tensor_wires(out_shape, WireVisibility::Intermediate);

        for i in 0..x_wires.wires.len() {
            let x_pos = self.alloc_wire();
            let x_neg = self.alloc_wire();

            // Constraint 1: x = x_pos - x_neg
            let x_lc = LinearCombination::single(FieldElement::one(), x_wires.wires[i]);
            let pos_minus_neg = LinearCombination {
                terms: vec![
                    (FieldElement::one(), x_pos),
                    (FieldElement::BN254_PRIME.sub(&FieldElement::one()), x_neg), // -1 * x_neg
                ],
                constant: FieldElement::zero(),
            };
            self.circuit.add_constraint(R1CSConstraint {
                a: pos_minus_neg,
                b: LinearCombination::constant(FieldElement::one()),
                c: x_lc,
                origin: self.origin_str("relu: x = x_pos - x_neg", i),
            });

            // Constraint 2: x_pos * x_neg = 0 (mutual exclusivity)
            self.circuit.add_constraint(R1CSConstraint {
                a: LinearCombination::single(FieldElement::one(), x_pos),
                b: LinearCombination::single(FieldElement::one(), x_neg),
                c: LinearCombination::constant(FieldElement::zero()),
                origin: self.origin_str("relu: x_pos * x_neg = 0", i),
            });

            // Constraint 3: out = x_pos
            self.circuit.add_constraint(R1CSConstraint {
                a: LinearCombination::single(FieldElement::one(), x_pos),
                b: LinearCombination::constant(FieldElement::one()),
                c: LinearCombination::single(FieldElement::one(), out_wires.wires[i]),
                origin: self.origin_str("relu: out = x_pos", i),
            });

            // Constraints 4+: bit decomposition to prove x_pos >= 0 and x_neg >= 0
            // For INT8 range [-128, 127], we need 8 bits per value.
            self.emit_range_check(x_pos, 8);
            self.emit_range_check(x_neg, 8);
        }

        out_wires
    }

    /// Emit range check constraints: prove that wire value fits in `bits` bits.
    /// Decomposes the value into individual bits and constrains each to be 0 or 1.
    fn emit_range_check(&mut self, wire: Wire, bits: u32) {
        let bit_wires: Vec<Wire> = (0..bits).map(|_| self.alloc_wire()).collect();

        // Each bit is 0 or 1: bit_i * (1 - bit_i) = 0
        for &bit_wire in &bit_wires {
            let bit_lc = LinearCombination::single(FieldElement::one(), bit_wire);
            let one_minus_bit = LinearCombination {
                terms: vec![(FieldElement::BN254_PRIME.sub(&FieldElement::one()), bit_wire)],
                constant: FieldElement::one(),
            };
            self.circuit.add_constraint(R1CSConstraint {
                a: bit_lc,
                b: one_minus_bit,
                c: LinearCombination::constant(FieldElement::zero()),
                origin: ConstraintOrigin {
                    source_span: Span::default(),
                    operation: "range_check: bit * (1 - bit) = 0".to_string(),
                    tensor_indices: None,
                },
            });
        }

        // Reconstruction: wire = sum(bit_i * 2^i)
        let reconstruction = LinearCombination {
            terms: bit_wires.iter().enumerate().map(|(i, &w)| {
                let coeff = FieldElement::from_int8(1).mul(
                    &FieldElement { limbs: [1u64 << i, 0, 0, 0] }
                );
                (coeff, w)
            }).collect(),
            constant: FieldElement::zero(),
        };
        self.circuit.add_constraint(R1CSConstraint {
            a: reconstruction,
            b: LinearCombination::constant(FieldElement::one()),
            c: LinearCombination::single(FieldElement::one(), wire),
            origin: ConstraintOrigin {
                source_span: Span::default(),
                operation: "range_check: reconstruction".to_string(),
                tensor_indices: None,
            },
        });
    }

    /// Compile a constant tensor into the circuit.
    /// Constants are embedded directly as known wire values.
    fn compile_constant(&mut self, value: &ConstValue, shape: &[usize]) -> TensorWires {
        let total_elems: usize = shape.iter().product();
        let mut wires = Vec::with_capacity(total_elems);

        match value {
            ConstValue::Int8Array(data) => {
                for &val in data {
                    let wire = self.alloc_wire();
                    // Wire has a fixed value — the prover must assign this exact value
                    self.circuit.set_constant_wire(wire, FieldElement::from_int8(val));
                    wires.push(wire);
                }
            }
            _ => panic!("non-integer constant in @zk_proof circuit"),
        }

        TensorWires {
            wires,
            shape: shape.to_vec(),
            visibility: WireVisibility::Intermediate,
        }
    }

    /// Compile weight access (self.W1).
    /// In weight_private mode, weights are private inputs.
    /// In input_private mode, weights are public inputs.
    fn compile_weight_access(
        &mut self,
        _object: &TypedExpr,
        field: Symbol,
        shape: &[usize],
    ) -> TensorWires {
        let visibility = match self.mode {
            ZkMode::WeightPrivate | ZkMode::FullPrivate | ZkMode::ArchitectureAttestation => {
                WireVisibility::Private
            }
            ZkMode::InputPrivate => WireVisibility::Public,
        };

        // Check if we already have wires for this weight
        if let Some(existing) = self.var_to_wires.get(&field) {
            return existing.clone();
        }

        let tensor_wires = self.alloc_tensor_wires(shape, visibility);

        // If M52 provides known weight values, set them as constants
        if let Some(weight_map) = &self.weights {
            let field_name = self.interner.resolve(field);
            if let Some(values) = weight_map.get_int8(field_name) {
                for (i, &val) in values.iter().enumerate() {
                    self.circuit.set_constant_wire(
                        tensor_wires.wires[i],
                        FieldElement::from_int8(val),
                    );
                }
            }
        }

        self.var_to_wires.insert(field, tensor_wires.clone());
        tensor_wires
    }

    /// Allocate wires for a tensor with given shape.
    fn alloc_tensor_wires(
        &mut self,
        shape: &[usize],
        visibility: WireVisibility,
    ) -> TensorWires {
        let total: usize = shape.iter().product();
        let wires: Vec<Wire> = (0..total).map(|_| self.alloc_wire()).collect();

        match visibility {
            WireVisibility::Public => {
                self.circuit.public_inputs.extend_from_slice(&wires);
            }
            WireVisibility::Private => {
                self.circuit.private_inputs.extend_from_slice(&wires);
            }
            WireVisibility::Intermediate => {}
        }

        TensorWires {
            wires,
            shape: shape.to_vec(),
            visibility,
        }
    }

    fn alloc_wire(&mut self) -> Wire {
        let w = Wire(self.next_wire);
        self.next_wire += 1;
        self.circuit.num_wires = self.next_wire;
        w
    }

    fn origin(&self, expr: &TypedExpr, elem_idx: usize) -> ConstraintOrigin {
        ConstraintOrigin {
            source_span: expr.span,
            operation: format!("{:?}", expr.kind),
            tensor_indices: Some(vec![elem_idx]),
        }
    }

    fn origin_str(&self, desc: &str, elem_idx: usize) -> ConstraintOrigin {
        ConstraintOrigin {
            source_span: Span::default(),
            operation: desc.to_string(),
            tensor_indices: Some(vec![elem_idx]),
        }
    }
}
```

### 3.3 Constraint Cost Analysis

| Operation | Shape | Mul Constraints | Add Constraints (free) | Range Checks |
|-----------|-------|----------------|----------------------|--------------|
| `matmul(A, B)` | A:[m,k], B:[k,n] | m * n * k | m * n * (k-1) | 0 |
| `a + b` | [n] | 0 | n | 0 |
| `a * b` | [n] | n | 0 | 0 |
| `relu(x)` | [n] | n (pos*neg=0) | n (decomposition) | 2n (pos+neg) |
| `reshape` | any | 0 | 0 | 0 |
| `transpose` | any | 0 | 0 | 0 |

**Example: 2-layer MLP (784 -> 128 -> 10):**
- Layer 1 matmul: 784 * 128 = 100,352 mul constraints
- Layer 1 bias add: 128 free constraints
- Layer 1 ReLU: 128 mul + 256 range checks (8 bits each = 2,048 bit constraints)
- Layer 2 matmul: 128 * 10 = 1,280 mul constraints
- Layer 2 bias add: 10 free constraints
- **Total: ~103,808 constraints** (proving time ~2-5 seconds on commodity hardware)

### 4.1 R1CS to Halo2 Translation

The Halo2 proving system uses a different constraint format (PLONKish arithmetic) than R1CS. The `Halo2Backend` translates R1CS constraints into Halo2 custom gates.

```rust
    pub verification_key: Vec<u8>,

    pub circuit_bytes: Vec<u8>,

/// A generated proof.

    /// The proof bytes (SNARK proof).
    pub proof_bytes: Vec<u8>,

    /// Public inputs used during proof generation.
    pub public_inputs: Vec<FieldElement>,

    /// Public outputs (model predictions).
    pub public_outputs: Vec<FieldElement>,

    /// Proof generation time in milliseconds.

}

impl Halo2Backend {
    /// Compile R1CS circuit to Halo2 circuit and generate keys.
    pub fn compile(&self, r1cs: &R1CSCircuit) -> Result<ZkArtifacts, ZkError> {
        // Step 1: Determine table size
        let k = (r1cs.constraints.len() as f64).log2().ceil() as u32 + 1;

        // Step 2: Build Halo2 circuit from R1CS
        let halo2_circuit = self.r1cs_to_halo2(r1cs, k)?;

        // Step 3: Generate proving and verification keys (trusted setup for BN254,
        // transparent for IPA/Halo2)
        let params = self.generate_params(k)?;
        let pk = self.keygen_pk(&params, &halo2_circuit)?;
        let vk = self.keygen_vk(&params, &halo2_circuit)?;

        Ok(ZkArtifacts {
            proving_key: self.serialize_pk(&pk),
            verification_key: self.serialize_vk(&vk),
            circuit_bytes: self.serialize_circuit(&halo2_circuit),
        })
    }

    /// Generate a proof for a specific witness (input + weights).
    pub fn prove(
        &self,
        artifacts: &ZkArtifacts,
        witness: &Witness,
    ) -> Result<ZkProof, ZkError> {
        let start = std::time::Instant::now();

        let circuit = self.deserialize_circuit(&artifacts.circuit_bytes)?;
        let pk = self.deserialize_pk(&artifacts.proving_key)?;

        // Assign witness values to the circuit
        let assigned_circuit = self.assign_witness(&circuit, witness)?;

        // Generate the proof
        let proof_bytes = self.create_proof(&pk, &assigned_circuit)?;

        Ok(ZkProof {
            proof_bytes,
            public_inputs: witness.public_inputs.clone(),
            public_outputs: witness.public_outputs.clone(),
            proving_time_ms: start.elapsed().as_millis() as u64,
        })
    }

    /// Verify a proof against the verification key.
    pub fn verify(
        &self,
        vk_bytes: &[u8],
        proof: &ZkProof,
    ) -> Result<bool, ZkError> {
        let vk = self.deserialize_vk(vk_bytes)?;

        // Verify the SNARK proof
        let valid = self.verify_proof(
            &vk,
            &proof.proof_bytes,
            &proof.public_inputs,
            &proof.public_outputs,
        )?;

        Ok(valid)
    }

    /// Translate R1CS constraints into Halo2 custom gates.
    fn r1cs_to_halo2(
        &self,
        r1cs: &R1CSCircuit,
        k: u32,
    ) -> Result<Halo2Circuit, ZkError> {
        // Each R1CS constraint A * B = C becomes a Halo2 custom gate.
        // Halo2 uses a table layout where each row contains advice (witness),
        // instance (public input), and fixed (constant) columns.

        // Allocate columns:
        // - One advice column per wire (up to a reasonable limit)
        // - Instance columns for public inputs/outputs
        // - Fixed columns for constants

        let num_advice = std::cmp::min(r1cs.num_wires as usize, 256);
        let num_instance = (r1cs.public_inputs.len() + r1cs.public_outputs.len())
            .next_power_of_two();

        // For large circuits, wires are packed across multiple rows
        let wires_per_row = num_advice;
        let num_rows = 1u64 << k;
```

### 4.2 Witness Generation

```rust
/// A witness assigns concrete values to every wire in the circuit.
pub struct Witness {
    /// Value for each wire, indexed by Wire ID.
    pub assignments: Vec<FieldElement>,

    /// Public inputs (subset of assignments, exposed to verifier).
    pub public_inputs: Vec<FieldElement>,

    /// Public outputs (subset of assignments, exposed to verifier).
    pub public_outputs: Vec<FieldElement>,
}

/// Generates a witness from concrete input tensors and model weights.
pub struct WitnessGenerator {
    /// The compiled circuit (for wire-to-variable mapping).
    circuit: R1CSCircuit,

    /// ZK privacy mode.
    mode: ZkMode,
}

impl WitnessGenerator {
    /// Generate witness for a forward pass.
    pub fn generate(
        &self,
        input: &[i8],     // flattened input tensor
        weights: &HashMap<String, Vec<i8>>,  // model weights by name
    ) -> Result<Witness, ZkError> {
        let mut assignments = vec![FieldElement::zero(); self.circuit.num_wires as usize];

        // Step 1: Assign input values to input wires
        for (i, &val) in input.iter().enumerate() {
            let wire = self.circuit.input_wires()[i];
            assignments[wire.0 as usize] = FieldElement::from_int8(val);
        }

        // Step 2: Assign weight values to weight wires
        for (name, values) in weights {
            let weight_wires = self.circuit.weight_wires(name);
            for (i, &val) in values.iter().enumerate() {
                assignments[weight_wires[i].0 as usize] = FieldElement::from_int8(val);
            }
        }

        // Step 3: Propagate through the circuit to compute intermediate wire values
        // Process constraints in topological order
        for constraint in &self.circuit.constraints {
            self.propagate_constraint(constraint, &mut assignments)?;
        }

        // Step 4: Classify public/private based on mode
        let public_inputs = self.circuit.public_inputs.iter()
            .map(|w| assignments[w.0 as usize])
            .collect();
        let public_outputs = self.circuit.public_outputs.iter()
            .map(|w| assignments[w.0 as usize])
            .collect();

        // Step 5: Verify all constraints are satisfied
        self.verify_witness(&assignments)?;

        Ok(Witness {
            assignments,
            public_inputs,
            public_outputs,
        })
    }

    /// Verify that the witness satisfies all R1CS constraints.
    fn verify_witness(&self, assignments: &[FieldElement]) -> Result<(), ZkError> {
        for (i, constraint) in self.circuit.constraints.iter().enumerate() {
            let a_val = self.evaluate_lc(&constraint.a, assignments);
            let b_val = self.evaluate_lc(&constraint.b, assignments);
            let c_val = self.evaluate_lc(&constraint.c, assignments);

            let ab = a_val.mul(&b_val);
            if ab != c_val {
                return Err(ZkError::WitnessViolation {
                    constraint_index: i,
                    origin: constraint.origin.clone(),
                    expected: c_val,
                    got: ab,
                });
            }
        }
        Ok(())
    }

    fn evaluate_lc(&self, lc: &LinearCombination, assignments: &[FieldElement]) -> FieldElement {
        let mut result = lc.constant;
        for &(coeff, wire) in &lc.terms {
            result = result.add(&coeff.mul(&assignments[wire.0 as usize]));
        }
        result
    }
}
```

### 6.1 ZkValidator Pass

New semantic pass: `crates/nsl-semantic/src/zk_check.rs`

```rust
/// Validates that @zk_proof functions use only circuit-compatible operations.
pub struct ZkValidator<'a> {
    interner: &'a Interner,
    diagnostics: &'a mut Vec<Diagnostic>,

    /// Set of allowed operations in ZK circuits.
    allowed_ops: HashSet<&'static str>,

    /// Set of allowed dtypes in ZK circuits.
    allowed_dtypes: HashSet<DType>,
}

impl<'a> ZkValidator<'a> {
    pub fn new(interner: &'a Interner, diagnostics: &'a mut Vec<Diagnostic>) -> Self {
        let allowed_ops: HashSet<&str> = [
            "matmul", "relu", "max", "min",
            "reshape", "transpose", "concat", "slice",
            "sum", "mean",
            "fixed_matmul", // fixed-point matmul
        ].into_iter().collect();

        let allowed_dtypes: HashSet<DType> = [
            DType::Int8, DType::Int16, DType::Int32,
            DType::Uint8, DType::Uint16, DType::Uint32,
        ].into_iter().collect();

        Self { interner, diagnostics, allowed_ops, allowed_dtypes }
    }

    /// Validate a @zk_proof function.
    pub fn validate_zk_function(&mut self, fn_def: &FnDef) {
        // Check 1: Return type must be integer or fixed-point
        self.check_dtype(&fn_def.return_type, fn_def.span);

        // Check 2: All parameter types must be integer or fixed-point
        for param in &fn_def.params {
            self.check_dtype(&param.ty, param.span);
        }

        // Check 3: No disallowed operations in function body
        self.check_body(&fn_def.body);

        // Check 4: No dynamic control flow
        self.check_no_dynamic_control_flow(&fn_def.body);

        // Check 5: Function must be @pure (compose with M51)
        // @zk_proof implies @pure — no IO, no Random, no Mutation, no Communication
    }

    fn check_body(&mut self, block: &Block) {
        for stmt in &block.stmts {
            self.check_stmt(stmt);
        }
    }

    fn check_stmt(&mut self, stmt: &Stmt) {
        match &stmt.kind {
            StmtKind::Expr(expr) => self.check_expr(expr),
            StmtKind::Let { value, .. } => self.check_expr(value),
            StmtKind::Return(Some(expr)) => self.check_expr(expr),
            StmtKind::If { condition, .. } => {
                // Data-dependent if is not allowed
                if self.is_data_dependent(condition) {
                    self.diagnostics.push(Diagnostic::error(
                        "@zk_proof does not allow data-dependent control flow".to_string(),
                        stmt.span,
                    ));
                }
            }
            StmtKind::For { .. } => {
                // Variable-length loops are not allowed
                // Fixed-length loops (compile-time known bounds) are OK
                self.check_fixed_loop(stmt);
            }
            _ => {}
        }
    }

    fn check_expr(&mut self, expr: &Expr) {
        match &expr.kind {
            ExprKind::Call { func, .. } => {
                let name = self.interner.resolve(*func);
                if !self.allowed_ops.contains(name) {
                    self.diagnostics.push(Diagnostic::error(
                        format!(
                            "operation '{}' is not compatible with @zk_proof — \
                             see docs/zk-compatible-ops.md",
                            name
                        ),
                        expr.span,
                    ));
                }
            }
            ExprKind::BinaryOp { op, left, right } => {
                // +, -, * are OK. / is not (division in finite fields is expensive).
                if matches!(op, BinOp::Div) {
                    self.diagnostics.push(Diagnostic::error(
                        "division is not directly supported in @zk_proof — \
                         use multiplication by modular inverse instead".to_string(),
                        expr.span,
                    ));
                }
                self.check_expr(left);
                self.check_expr(right);
            }
            _ => {}
        }
    }

    fn check_dtype(&mut self, ty: &Type, span: Span) {
        if let Type::Tensor { dtype, .. } = ty {
            if !self.allowed_dtypes.contains(dtype) {
                self.diagnostics.push(Diagnostic::error(
                    format!(
                        "@zk_proof requires integer types (int8, int16, int32), got {:?} — \
                         quantize the model to INT8 first (M35)",
                        dtype
                    ),
                    span,
                ));
            }
        }
    }

    fn is_data_dependent(&self, _condition: &Expr) -> bool {
        // An if-condition is data-dependent if it references any tensor value
        // (not just compile-time constants or loop indices).
        // Conservative: any condition that isn't a compile-time constant is data-dependent.
        true // conservative default — refined during implementation
    }

    fn check_fixed_loop(&mut self, stmt: &Stmt) {
        // A for loop is allowed if the iteration count is known at compile time.
        // e.g., `for i in range(10)` is OK, `for i in range(len(x))` is not.
        if let StmtKind::For { iter, .. } = &stmt.kind {
            if !self.is_compile_time_bound(iter) {
                self.diagnostics.push(Diagnostic::error(
                    "@zk_proof requires compile-time-known loop bounds".to_string(),
                    stmt.span,
                ));
            }
        }
    }

    fn is_compile_time_bound(&self, _iter: &Expr) -> bool {
        // Check if the loop bound is a literal or compile-time constant
        false // conservative default — refined during implementation
    }
}
```

## Section 10: Deliverables

1. `@zk_proof` decorator with four privacy modes (weight_private, input_private, full_private, architecture_attestation)
2. `ZkValidator` semantic pass — validates circuit-compatible operations, integer types, no dynamic control flow
3. `CircuitCompiler` — expression DAG to R1CS constraint system for matmul, add, mul, relu, reshape, transpose
4. `FieldElement` — finite field arithmetic over BN254 (and BLS12-381, Goldilocks stubs)
5. `Halo2Backend` — R1CS to Halo2 circuit, proof generation, verification (via `halo2_proofs` crate)
6. `WitnessGenerator` — maps concrete input/weight tensors to field-element witness
7. `SolidityEmitter` — generates Solidity verification contract for on-chain proof checking
8. CLI: `nsl build --zk-circuit`, `nsl zk prove`, `nsl zk verify`, `nsl zk stats`
9. Integration with M52: weight sparsity reduces constraint count proportionally
10. Integration with M35: INT8 quantized models are the primary proof target

## Out of Scope

- Non-integer arithmetic in circuits (f32/f16/bf16 — these require floating-point-to-circuit emulation, which is impractical for large models)
- ZK circuits for backward passes / training (only forward inference is circuit-compiled)
- Recursive proof composition (proving a proof-of-proof — possible with Halo2 but deferred)
- GPU-accelerated proving (proving runs on CPU; GPU proof generation is a future optimization)
- Circuit optimization passes (constant gate folding, common subexpression elimination in the circuit — future work)
- ZK-friendly activation functions (e.g., square-based activations instead of ReLU — these are model architecture changes, not compiler changes)
- Plonky3 backend implementation (stub only — full implementation deferred to post-v1.0)
- Move/Solana verification contract generation (Solidity/EVM only in v1.0)
- Proof aggregation (batching multiple proofs into one — requires recursive SNARKs)
- Formal verification of circuit-model equivalence (empirically tested, not formally proven)

## Success Criteria

- Tiny 2-layer MLP (784->128->10, INT8): compiles to R1CS circuit with ~104k constraints
- Proof generation: completes in < 10 seconds on commodity hardware (8-core CPU)
- Proof verification: completes in < 5 ms
- Proof size: < 1 KB (Halo2/BN254)
- Wrong-output proof: verification returns false (soundness guarantee)
- Wrong-weight proof: verification returns false (weight integrity)
- Solidity contract: compiles with solc 0.8.20+, verifies proof on-chain for < 500k gas
- Float model with @zk_proof: clear compile error with quantization guidance
- Softmax in @zk_proof: clear compile error with alternative suggestions
- M52 sparsity: 50% zero weights -> ~50% fewer multiplication constraints
- All existing tests pass unchanged — @zk_proof is additive functionality
