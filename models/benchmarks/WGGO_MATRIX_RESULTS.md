# WGGO packed-pretraining validation matrix — results (2026-07-14)

Follow-up to `WGGO_ALL6_RESULTS.md` (PR #368), run on the PR #369 branch
(deferral closures: Tier-B tile-skip promoted, per-document RoPE reset,
GPU moment-cast envelope, gradient device-alignment, `NSL_PHASE_TIMING`).

**Hardware/software.** RTX 5070 Ti (16 GB, sm_120 — now auto-detected:
`[csha] gpu spec: RTX-5070-Ti`, smem negotiation reads the honest
`smem_208kb_exceeds_116kb` instead of the old A100-fallback 180 kb),
CUDA 13.x, release build, `nsl run --source-ad --pretrain-optimized`,
frozen binary for the whole matrix.

**Methodology.** Autogenerated GQA packed-LM configs (base: d_model 512,
6 blocks, 8Q/2KV hd 64, FFN 2048, vocab 4096, seq 1024, batch 4, accum 4,
192-token docs, AdamW + grad_clip 1.0). Per config: **5 repeats** (3 for
A/B pairs), 16 micro-batches each, first 2 dropped as JIT/alloc warmup →
n = 39–65 timed micro-batches per cell (126 for the 500M model, 125 for
the long run). Medians with IQR and stdev. `NSL_PHASE_TIMING=1` gives the
forward/backward/optimizer wall-clock split (device-synced clock reads).
VRAM/util from 0.5 s `nvidia-smi` polls. **Useful tok/s counts only
positions with label ≠ −100** (excludes EOS separators and per-row final
positions, computed exactly from the doc pattern) — reported next to raw
positions/s. Note: micro-batch dt distributions are bimodal at accum > 1
(every 4th micro-batch carries the ~160 ms optimizer step) — that is the
~155 ms IQR visible across most rows; medians land on non-step
micro-batches.

## Headline results

1. **Fused packed attention: 1.41–1.51× end-to-end** vs the decomposed
   fallback (kill-switch A/B), losses equal to 4 decimals. The win lives
   mostly in the backward (LSE-saved fused backward vs decomposed
   adjoints: 1915 vs 2800 ms at base).
2. **Tier-B tile-skip (newly promoted) cuts the fused FORWARD 1.45–2.63×**
   (2.63× at seq 4096: 433 vs 1139 ms) at **identical losses** in every
   pair. Whole-step effect is 1.04–1.12× because the backward dominates.
3. **The backward is 9–50× the forward** everywhere (base: 172 vs
   1916 ms; 500M: 148 ms vs 7.4 s). GPU util 9–36 %: absolute GPU-busy
   time grows far slower than step time as attention work scales — the
   decomposed backward is host-glue-bound, not FLOP-bound. This is THE
   perf lever for everything below 1 B params.
4. **Document density swings throughput 6×** at fixed geometry: 32-token
   docs → 1222 ms/mb (3243 useful tok/s), single-doc rows → 7449 ms/mb
   (549 tok/s). Causal-within-document attention means denser packing =
   strictly less attention work; Tier-B converts the boundary structure
   into skipped tiles on top.
5. **500M (503M params) trains on this 16 GB card**: 7.69 s/mb median,
   loss 10.82 → 4.65 over 64 micro-batches, 15.6 GiB peak, 30 % util.
   **1.04 B does NOT fit** — three-way OOM characterization below.
6. Repeatability: per-config stdev 75–100 ms on ~2–8 s micro-batches
   (≈2–4 %); identical configs across repeats agree to <0.5 %.

## Full matrix

(see `WGGO_MATRIX_TABLE.md` for the machine-generated table with all 31
rows; excerpts below)

### A. Sequence length (fixed 8192 tokens/micro-batch)

| seq×batch | med ms/mb | fwd ms | bwd ms | useful tok/s |
|---|---|---|---|---|
| 512×16 | 3641 | 304 | 3302 | 2234 |
| 1024×8 | 4092 | 342 | 3715 | 1990 |
| 2048×4 | 4721 | 373 | 4312 | 1725 |
| 4096×2 | 5862 | 433 | 5386 | 1390 |
| 8192×1 | 7753 | 407 | 7258 | 1051 |

Forward grows only 1.3× across a 16× seq range (flash + tile-skip); the
2.1× step-time growth is all in the decomposed backward. The residual
per-mb time not covered by fwd+bwd+opt is ≈0 everywhere (≤48 ms even at
seq 8192 where the DataLoader builds a 268 MB dense mask — mask cost is
real but small).

### B/C. Batch and accumulation (seq 1024)

Batch scales near-linearly (591/1126/2105/4074 ms for b=1/2/4/8) with
per-token throughput *improving* with batch (1734 → 2011 tok/s).
Accumulation is free per micro-batch (2146/2111/2108 ms at accum 1/4/8);
the ~162 ms optimizer step amortizes across the window.

### D. Head geometry (d_model 512 fixed)

| geometry | med ms/mb | useful tok/s |
|---|---|---|
| 4Q/1KV hd128 (MQA-wide) | 2024 | 2011 |
| 8Q/1KV hd64 (MQA) | 2093 | 1945 |
| 8Q/2KV hd64 (GQA-4, base) | 2109 | 1930 |
| 8Q/8KV hd64 (MHA) | 2130 | 1911 |
| 16Q/4KV hd32 | 2478 | 1643 |

Fewer, wider heads win (hd128 fastest, hd32 slowest — more per-head
launches). KV-head count barely matters at this scale; MHA costs +12 %
VRAM (6.6 vs 5.9 GiB) for the bigger KV.

### E. Document-boundary patterns (base geometry)

| docs | med ms/mb | useful tok/s | GPU util |
|---|---|---|---|
| 32-token | 1222 | 3243 | 34 % |
| 64-token | 1413 | 2852 | 29 % |
| 192-token | 2136 | 1906 | 21 % |
| 512-token | 4154 | 983 | 12 % |
| realistic (lognormal 23–956) | 3817 | 1069 | 13 % |
| single doc/row | 7449 | 549 | 9 % |

The "realistic" mixture (16 lognormal-drawn lengths, median ≈180) lands
between the 192 and 512 fixed patterns, with the highest variance
(IQR 613 ms — long docs make uneven micro-batches).

### G. Tier-B tile-skip (NSL_SDPA_TIER_B_DISABLE=1) — first production measurement

| config | fwd on/off (ms) | fwd gain | step gain | loss on/off |
|---|---|---|---|---|
| docs 192, seq 1024 | 171 / 248 | 1.45× | 1.04× | 6.4362 / 6.4362 |

| docs 192, seq 4096 | 433 / 1139 | **2.63×** | 1.12× | 6.3471 / 6.3471 |

Losses identical in every pair — E2E confirmation of the kernel-level
bitwise parity gate that justified promotion.

### L. Long run (128 micro-batches, base geometry)

2106 ms/mb median — **7 % faster than the PR #368-era binary (2264 ms)**
on the same geometry, from Tier-B + gradient device-alignment. Loss
8.445 → 0.0231 (the #368 run reached 0.0236; trajectories are not
bit-comparable because RoPE positions now reset per document — the
intended semantic change).

## 500M model (503M params: d1536, 18 blocks, 12Q/4KV hd128, FFN 6144, vocab 32k)

seq 512, batch 1, accum 8, realistic docs, 2 repeats × 64 micro-batches:

* med 7.69 s/mb (IQR 2.0 s — allocator pressure near the VRAM ceiling
  causes occasional multi-second stalls, max 24.6 s), fwd **148 ms** vs
  bwd **7.41 s**, optimizer step 13.8 s (FASE merge over 503M params).
* loss 10.822 → 4.654; VRAM peak 15.6 GiB; util 30 %; 67 useful tok/s.
* At seq 1024 it OOMs (first attempt died mid-forward at 15.8 GiB).

## 1B model (1.037B params: d2048, 22 blocks, 16Q/4KV hd128, FFN 8192): does NOT fit

Three configurations, three distinct walls, all at 15.7–15.8 GiB:

1. **AdamW, accum 8** — dies at optimizer-STATE allocation
   (`nsl_tensor_zeros_like`): FASE-Deferred AdamW state is
   weights + m + v + m_partial + v_partial = 5 × 4 B ≈ **20.7 GB > 16 GB**
   before the first forward.
2. **SGD+momentum, accum 8** — state (w + velocity + partials) fits
   allocation, dies in the first forward matmul.
3. **SGD+momentum, accum 1** — forward completes (11.7 GiB), dies in the
   backward weight-grad reduce: + in-flight gradients + saved
   activations crosses the ceiling.

The practical ceiling on a 16 GB card today is **~500M with AdamW**
(≈31 bytes/param all-in at seq 512). The levers that would move it, in
order of impact: (a) an end-to-end driver for reduced-precision moments —
the GPU cast envelope now works (PR #369) but neither WGGO (no
memory-budget input in prod) nor CPDT (evidence-gated without weights)
organically selects sub-32 moments on random init; (b) activation
checkpointing on the decorator-free path; (c) allocator trim/defrag near
the ceiling (the 500M IQR shows the cost of running at 95 % VRAM).

## Honest notes

* Phase timing was ON for all runs (3 extra device syncs per micro-batch
  + 2 per optimizer step); the syncs are ~µs against multi-second steps.
* The matrix's first 2 micro-batches per run (JIT + PTX + allocator
  warmup) are excluded from stats; the long-run row shows steady state.
* One `[512]`-shaped CPU f64→f32 gradient reconciliation per micro-batch
  remains (≈2 KB; visible under `NSL_RECONCILE_DEBUG=1`) — unidentified
  producer, negligible cost, tracked as a follow-up.
* Config generator + driver: `models/benchmarks/matrix_gen_model.py`,
  `matrix_configs.json` (this commit); raw per-run JSON is session-local.
